# 关键决策与待定项

> 目标：沉淀已确认决策，明确仍需拍板的问题。

## 已确认决策

- 运行环境优先本地单机
- 群聊入口优先飞书
- 默认不依赖 PostgreSQL/Redis
- Docker 为可选本地部署
- 高风险动作默认 HITL 审批
- 事件范围按 P0/P1/P2 分层推进
- 交互入口策略：`CLI` 作为主入口，`Web UI` 作为可选管理面板，`TUI` 暂不优先
- Web UI 技术路线：优先复用现有 `Next.js + React + Tailwind`，不在当前阶段切换到新前端框架

## 待定项（需要后续实现时确认）

- 规则热更新是否默认开启
- 回放接口的权限控制（本地模式下的安全边界）
- 默认事件保留天数是否固定 30 天
- `TUI` 是否在 P2 后进入路线图（取决于真实终端用户需求）
- Web UI 是否在后续独立成单独仓库（目前可先保留在 monorepo）

## 决策更新规则

- 每次决策变更需更新此文件
- 变更应同步到 `README.md` 与相关设计文档
- 重大变更建议附带迁移影响说明
