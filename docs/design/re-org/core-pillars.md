# Semibot 四大核心特性（重构对齐）

> 目标：将“快速执行、主动工作、互相协作、持续进化”落地为系统能力与可验证指标，作为重构设计的对齐基准。

## 1. 快速执行

**定义**：用户提出任务后，系统能够在短时间内完成执行或给出可行动结果。

**机制落点**
- 预热容器池 / 资源治理（降低冷启动）
- MCP 多协议连接池（减少外部工具调用延迟）
- 统一能力图（Capability Graph）快速路由
- 事件框架（Reflex Engine）减少链路开销

**可验证指标**
- P95 首次可用响应时间
- 平均任务完成时间（按任务类型分层）
- 冷启动占比

## 2. 主动工作

**定义**：系统能够在合适时机主动发现问题、提出问题或建议，并在授权后执行。

**机制落点**
- 统一事件框架（hooks/webhooks/heartbeat/cron → event）
- 触发规则引擎（去重、节流、风险分级、置信度阈值）
- HITL 审批与注意力预算
- 事件回放与审计

**可验证指标**
- 主动建议采纳率
- 主动询问噪声率（被忽略/拒绝比例）
- 事件触发重复率

## 3. 互相协作

**定义**：多个 Agent 能分工协作，完成复杂任务并汇总结果。

**机制落点**
- LangGraph Supervisor‑Worker 模式
- 任务分解与执行计划（ExecutionPlan）
- 共享记忆 + 任务队列
- 冲突仲裁与结果汇总策略

**可验证指标**
- 复杂任务的并行完成率
- 任务拆解后重试率
- 聚合结果准确率

## 4. 持续进化

**定义**：系统能够从成功执行中沉淀能力，形成可复用技能并持续提升。

**机制落点**
- 技能生命周期（安装/验证/激活/版本锁定/审计）
- 进化技能（evolved_skills）自动萃取
- 事件化进化流水线（候选→评分→审核→发布）
- 语义检索与技能复用

**可验证指标**
- 进化技能采用率
- 新技能质量评分分布
- 复用带来的成本下降比例

---

**说明**：本文件作为重构的目标基准，所有新模块或设计调整需明确对齐以上四个特性之一，并具备可观测指标。
