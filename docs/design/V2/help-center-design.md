# Help Center 设计（Web）

> 版本：1.0 | 日期：2026-03-01 | 状态：已落地首版

## 1. 目标

帮助系统用于解决三个问题：

1. 新用户不知道每个页面是做什么的。
2. 功能按钮和配置项的操作影响不明确。
3. 高价值配置（尤其 Gateway）缺少统一教程与排障路径。

## 2. 信息架构

- 入口位置：左侧导航栏最下方，固定入口 `Help Center`。
- 路由：`/help`。
- 页面结构：
  - 功能导航速览（对应左侧所有模块）
  - Hover 说明使用指南
  - Gateway 配置详细教程（重点）
  - Gateway FAQ（常见故障与处理）

## 3. Hover 说明规范

- 导航项：鼠标悬停显示“模块职责 + 适用场景”。
- 关键按钮：鼠标悬停显示“将刷新/修改什么数据”。
- 快捷入口：鼠标悬停显示“该入口可完成的目标”。

约束：

- 文案必须是“可执行说明”，避免仅重复按钮名称。
- 同一功能在不同页面的描述要一致。
- 中英日三语文案同步维护。

## 4. Gateway 教程范围

教程覆盖 Telegram / Feishu，按“从可用到可观测”的顺序组织：

1. 准备参数（token、webhook、agent、allowedChatIds）。
2. 新建 Gateway 实例并保存基础配置。
3. 设置 addressing/proactive/context 策略。
4. 执行连通性测试并确认 `ready`。
5. 配置第三方回调 URL 到 Runtime 接口。
6. 在事件中心与网关会话详情中验证入站和回传。

## 5. FAQ 最小集合

- 群里发消息机器人无响应。
- 消息未命中（allowedChatIds 或 agent 绑定问题）。
- token/secret 无效。
- 任务卡在审批。

## 6. 版本计划

- V1（当前）：导航入口 + 帮助中心页面 + Gateway 教程 + Hover 说明首批覆盖。
- V2：扩展到每个配置字段的上下文帮助（字段级 Tooltip）。
- V3：接入“帮助搜索 + 场景化引导流程（step-by-step wizard）”。
